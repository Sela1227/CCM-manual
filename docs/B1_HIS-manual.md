## 1. 系統概述

### 什麼是 HIS？

HIS（Hospital Information System）是醫院的核心資訊系統。本院的 HIS 整合了多個子系統，用同一個帳號登入就可以切換使用。

### 子系統總覽

| 子系統 | 功能說明 | 使用頻率 |
|--------|---------|---------|
| 門診系統 | 查門診病歷、看診紀錄 | ⭐⭐⭐ 每天 |
| 住院系統 | 查住院病歷、住院狀況 | ⭐⭐⭐ 每天 |
| 會診系統 | 開立會診、查詢會診結果 | ⭐⭐ 常用 |
| 檢查系統 | 排檢查、查檢查排程 | ⭐⭐ 常用 |
| PACS 系統 | 看影像（CT、MRI、X光等） | ⭐⭐ 常用 |
| 檢驗系統 | 查抽血報告、腫瘤指數 | ⭐⭐⭐ 每天 |
| 癌症總匯系統 | 查癌症病人整合資訊 | ⭐⭐⭐ 每天 |
| 化療藥物系統 | 查化療處方、療程 | ⭐⭐ 常用 |
| 放射線治療系統 | 查放療排程、進度 | ⭐ 需要時 |
| 期別系統 | 查詢/登錄癌症分期 | ⭐⭐ 收案時 |
| 癌計書系統 | 癌症治療計畫書 | ⭐⭐ 常用 |
| 多專科會議系統 | MDT 會議排程、紀錄 | ⭐ 每月 |

## 15. 常見問題排解

### Q: 系統登不進去？

A: 
1. 確認帳號密碼是否正確
2. 確認 Caps Lock 沒有開啟
3. 嘗試重新開機
4. 還是不行 → 聯絡資訊室

### Q: 找不到病人資料？

A:
1. 確認病歷號是否正確
2. 用身分證字號查（注意英文要大寫）
3. 確認病人是否在本院有就醫紀錄

### Q: 報告還沒出來？

A:
- 不同檢查出報告時間不同（參考各系統說明）
- 可以先跟病人說大概時間
- 急件可以聯絡相關單位

### Q: 系統很慢或當掉？

A:
1. 等待一下再試
2. 關掉不需要的視窗
3. 重新登入
4. 還是不行 → 聯絡資訊室

### Q: 操作上有問題不會用？

A:
1. 先問學姊
2. 查這份手冊
3. 還是不會 → 問資訊室

## 一、個管病患追蹤系統

### 這個系統是做什麼的？

這是個管師日常工作的核心平台，取代紙本或 Excel 的個案管理方式。每個在管的個案，追蹤記錄、照護狀態、聯絡紀錄都在這裡維護。

### 主要功能

- 個案清單：查看自己負責的所有在管個案
- 追蹤記錄：每次聯絡或訪視後的記錄輸入
- 狀態標記：在管、失聯、結案、死亡等狀態管理
- 提醒功能：追蹤到期提醒、回診提醒
- 指標統計：失聯率、追蹤完成率等基本統計

### 什麼時候會用到

| 情境 | 操作 |
|------|------|
| 每次電話追蹤後 | 記錄追蹤結果、更新聯絡日期 |
| 個案回診後 | 更新照護狀態 |
| 每月初 | 確認本月應追蹤名單 |
| 個案失聯 | 標記失聯日期、記錄嘗試聯絡次數 |

## 二、癌症資源中心系統

### 這個系統是做什麼的？

整合院內外的照護資源，讓個管師在轉介和查找資源時有統一的入口。不用每次都憑記憶或到處詢問。

- 轉介記錄：記錄已轉介個案的進度與結果
- 衛教材料庫：各癌別衛教單張、影片連結
- 補助資訊更新：定期維護最新的社福補助資訊

### 什麼時候會用到

| 情境 | 操作 |
|------|------|
| 個案需要轉介社工、心理師等 | 開立轉介、追蹤轉介進度 |
| 個案詢問補助或資源 | 查詢最新資源資訊 |
| 衛教時需要資料 | 下載或分享衛教材料 |

### 這個系統是做什麼的？

將肺癌的標準照護流程數位化，讓個管師可以清楚掌握每個肺癌個案「現在在哪個節點」、「下一步應該做什麼」、「有沒有照時程進行」。

### 適用對象

### 主要功能

- 路徑視覺化：以時間軸呈現個案的照護節點
- 待辦追蹤：每個節點是否如期完成的紀錄
- 偏差提醒：個案進度落後或偏離路徑時的警示

## 四、個管多專科會議系統

### 這個系統是做什麼的？

MDT 會議的前置作業和會後追蹤的數位化工具，讓個管師不需要用紙本或 Word 準備個案資料，改用統一格式輸入，會後決議也在同一個地方追蹤。

### 主要功能

- 個案報告準備：依照標準格式輸入個案資料、問題
- 會議紀錄：記錄各專科意見與團隊決議
- 決議追蹤：追蹤 MDT 決議的執行狀況
- 歷次會議查閱：查詢個案過去在 MDT 的討論紀錄

### 什麼時候會用到

| 時機 | 操作 |
|------|------|
| 會議前 1–2 天 | 輸入本次要討論的個案資料 |
| 會議中 | 記錄討論重點與決議 |
| 會議後 | 追蹤決議執行情形 |

## 五、病歷互審系統

### 這個系統是做什麼的？

個管師之間或個管師與主管之間，定期互相審閱照護記錄的品質，作為內部品質管理的工具。目的是找出記錄不完整或照護邏輯有疑問的地方，互相學習，不是互相找麻煩。

### 主要功能

- 審查任務分配：系統指定審閱對象與期限
- 審查表單：標準化的審閱項目與評分
- 回饋記錄：審閱意見與被審者回覆
- 趨勢統計：追蹤照護記錄品質的改善趨勢

### 注意事項

> 💡 新人在試用期間通常以「被審」為主，先理解系統如何運作。約在正式任用後 3–6 個月，才會開始參與互審工作。

## 系統學習建議順序

對新進個管師來說，這五個系統的學習建議依以下順序進行：

| 學習順序 | 系統 | 建議時機 |
|---------|------|---------|
| 1 | 個管病患追蹤系統 | 第 5–6 週（學追蹤管理時同步學） |
| 2 | 癌症資源中心系統 | 第 4 週（學轉介資源時同步了解） |
| 3 | 個管多專科會議系統 | 第 8 週（學 MDT 參與時同步學） |
| 4 | 肺癌臨床路徑 | 第 9–10 週（學肺癌癌別時同步學） |
| 5 | 病歷互審系統 | 正式任用後 3 個月 |
