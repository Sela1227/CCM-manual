# 癌症個管師　工具篇

**彰濱秀傳癌症中心｜癌症個案管理師指導書 · 最終修訂版**

---

這份檔案彙整個管師會用到的全部資訊系統——醫院 HIS 各子系統、院內自製工具（個管追蹤系統、MDT 會議系統、抗癌藥物速查、病歷互審系統）、以及通用工具（排程、LINE、信箱、雲端、視訊）。

新人入職第二週開始用本篇對照系統實作。資深個管師日常查操作步驟。

## 1. 工具總覽（軟體上手指南）

這部分教你怎麼使用工作上會用到的系統。剛開始可能會覺得複雜，但用幾次就會熟了。

---

### 系統總覽

本院的系統主要是 **HIS 醫療資訊系統**，裡面整合了很多子系統。個管師不需要操作癌登系統（由癌登師負責）。

#### HIS 系統與子系統

| 子系統 | 你用它來做什麼 | 使用頻率 |
|--------|---------------|---------|
| **門診系統** | 查門診病歷、看診紀錄 | 每天 |
| **住院系統** | 查住院病歷、住院狀況 | 每天 |
| **會診系統** | 開立會診、查詢會診結果 | 常用 |
| **檢查系統** | 排檢查、查檢查排程 | 常用 |
| **PACS 系統** | 看影像（CT、MRI、X光等） | 常用 |
| **檢驗系統** | 查抽血報告、腫瘤指數 | 每天 |
| **癌症總匯系統** | 查癌症病人整合資訊 | 每天 |
| **化療藥物系統** | 查化療處方、療程 | 常用 |
| **放射線治療系統** | 查放療排程、進度 | 需要時 |
| **期別系統** | 查詢/登錄癌症分期 | 收案時 |
| **癌計書系統** | 癌症治療計畫書 | 常用 |
| **多專科會議系統** | MDT 會議排程、紀錄 | 每月 |

> 💡 這些子系統都在 HIS 裡面，用同一個帳號登入就可以切換使用。

### 15. 常見問題排解

#### Q: 系統登不進去？

A: 
1. 確認帳號密碼是否正確
2. 確認 Caps Lock 沒有開啟
3. 嘗試重新開機
4. 還是不行 → 聯絡資訊室

#### Q: 找不到病人資料？

A:
1. 確認病歷號是否正確
2. 用身分證字號查（注意英文要大寫）
3. 確認病人是否在本院有就醫紀錄

#### Q: 報告還沒出來？

A:
- 不同檢查出報告時間不同（參考各系統說明）
- 可以先跟病人說大概時間
- 急件可以聯絡相關單位

#### Q: 系統很慢或當掉？

A:
1. 等待一下再試
2. 關掉不需要的視窗
3. 重新登入
4. 還是不行 → 聯絡資訊室

#### Q: 操作上有問題不會用？

A:
1. 先問學姊
2. 查這份手冊
3. 還是不會 → 問資訊室

### 三、病人管理

#### 3-1 新增病人
1. 點左側「病人管理」
2. 右上角「新增病人」按鈕
3. 填寫基本資料：姓名、病歷號、性別、出生日期、癌別
4. 點「儲存」

#### 3-2 查詢病人
- 上方搜尋列輸入**姓名**或**病歷號**即時篩選
- 可依癌別、狀態（在案/結案/已歿）篩選
- 多重癌病人（同病歷號有多筆記錄）會顯示**「多重癌」**紅色標籤

#### 3-3 病人詳情
點擊任一病人進入詳情頁，共 7 個分頁：

| 分頁 | 內容 |
|------|------|
| 總覽 | 基本資訊摘要、最近事件 |
| 基本資料 | 個人基本資訊編輯 |
| 分期 | 臨床/病理分期（cTNM/pTNM） |
| 收案/癌別 | 收案資訊、癌別特有欄位 |
| 事件記錄 | 所有治療事件（手術/化療/放療等）的新增與管理 |
| 指標 | 這位病人相關的品質指標達成狀況 |

4. 填寫相關欄位後儲存

> **MDT 相關事件**：若某事件需要提交 MDT 討論，勾選「需要 MDT 討論」flag，建立 MDT 會議時系統會自動偵測並列入必要提報名單。

### 四、多專科會議（MDT）

#### 4-1 行事曆
- **實色徽章**：已建立的會議，點擊查看詳情
- **虛線徽章**：系統根據排程原則自動推算的建議日期，點擊直接建立會議
- **統計 tab**：各癌別群組討論量、每月趨勢、必要提報類別分布

#### 4-2 建立會議
1. 點行事曆上的虛線日期，或右上角「新建會議」
2. 選擇**癌別群組**（乳癌/胸腔癌/頭頸癌…）
3. 系統自動帶入：建議排程原則、負責個管師、會議地點、預設時間
4. 日期/時間/地點皆可手動修改（實際借到的場地可能不同）
5. 點「建立會議」

#### 4-3 複製會議
在已建立的會議詳情面板：
1. 點「**複製**」按鈕
2. 選擇目標日期（通常是下個月同一癌別的日期）
3. 確認複製：病人清單會複製過去，討論要點/結論清空讓個管師重新填寫

#### 4-4 進入會議室
在會議詳情面板點「**進入會議 →**」，進入會議室有 5 個區塊：

| Section | 說明 | 自動來源 |
|---------|------|---------|
| ⭐ 新個案討論 | 上次同癌別 MDT 後新收的病人 | 系統自動列出，個管師勾選 |
| 📈 前期追蹤 | 上次 MDT 有結論、這次要回報 | 系統自動列出 |
| ⚠️ 必要提報 | 符合基準2.4八類條件的病人 | 系統自動偵測（⚡標示） |
| 📋 醫療小組 | 個管師回覆意見追蹤 | 有醫療小組事件卡的病人 |
| 👤 特殊議程 | 臨時加入 | 手動搜尋加入 |

**手動加入病人：**
- 點各 Section 右上的「加入病人」
- 輸入姓名或病歷號搜尋
- 選擇要加入哪個 Section
- 必要提報/醫療小組類型會同步幫病人建立事件卡

**填寫討論內容：**
- 點擊病人卡片展開
- 自動帶入治療記錄（手術/化療/放療）
- 填寫：影像結果、病理結果、討論要點、**結論與建議**（有結論邊框轉綠）、後續追蹤計畫

#### 4-5 會後填寫
點 Header「**會後填寫**」按鈕：
- 一次看所有討論個案，補填摘要/結論/追蹤
- **帶入下次追蹤 toggle**：關閉後，此個案下次不會自動出現在前期追蹤名單
- 儲存時自動更新病人的 MDT 事件卡結論

#### 4-6 完成會議
點「**完成會議**」：
1. 所有討論內容自動儲存
2. 有結論的病人自動新增 `MDT 結論` 事件卡
3. 已標記 `需要 MDT 討論` 的事件卡自動標記為已討論
4. 會議狀態改為「已完成」

#### 4-7 產出功能
Header 右側有四個產出按鈕，前方五顆色票可選配色主題：

| 按鈕 | 說明 |
|------|------|
| **HTML** | 下載 HTML 投影片，瀏覽器直接開啟播放 |
| **LINE** | 產生會議通知文字，自動複製到剪貼簿 |
| **DOCX** | 下載 Word 格式會議記錄 |
| **PPTX** | 下載 PowerPoint 簡報 |

**HTML 投影片操作：**

- `Home` / `End` 跳到第一/最後一頁

### 二、開會前的準備

主畫面點「**建立新會議**」→ 選擇會議日期、時間、地點 → 勾選參與的癌別。

!!! tip "典型情境"

| 區塊 | 用途 |
|------|------|
| **前期追蹤** | 上次會議討論過、本次要回報進度的個案 |

如果不想要某個案被自動帶入，在上次會議的會後填寫時把該個案的「下次追蹤」設為「**結案**」即可。

### 八、常見問題

**Q：同事改的我看不到？**

存檔後 1-2 分鐘內推上 NAS，對方下次登入會拉下來。手動觸發：設定 → 備份 → 立即同步。

**Q：不小心關了瀏覽器，資料會不見嗎？**

不會。資料存在瀏覽器（localStorage），關掉重開仍在。但**清快取會清掉本機資料**，養成習慣定期確認 NAS 是否有備份。

**Q：為什麼某個個案沒被自動帶入下次？**

兩種可能：(1) 上次會議的「會後填寫」把它設為「結案」；(2) 該個案不在「前期追蹤」區裡。

**Q：投影片字太小看不清？**

HTML 投影片內有放大鏡功能（R 鍵或點擊圖片）；若整體字小，可用瀏覽器縮放（Ctrl + 滾輪）。

**Q：想印投影片？**

選**高對比**配色（黑灰白）再產出 → 用瀏覽器列印或產 PPTX 後在 PowerPoint 列印。

**Q：想復原一個誤刪的會議？**

如果 90 天內發現誤刪，且 NAS 上還有 tombstone 標記未清除，聯絡資訊人員手動處理。從前端介面無法直接復原，所以刪除前請務必確認。

    此類藥物（如 pembrolizumab、nivolumab、atezolizumab 等）條文最長，含大量 PD-L1 門檻表格。系統已手工整理為逐癌切片，選擇癌別後只顯示該癌別相關條件，大幅降低查閱難度。

---

### 藥物類型說明

個管師在協助病人了解治療方案時，可參考以下類型說明：

| 類型 | 說明 | 常見藥物 |
|------|------|---------|
| **化學治療** | 細胞毒殺性藥物，干擾癌細胞分裂 | Paclitaxel、Carboplatin、Gemcitabine |
| **標靶治療** | 針對特定基因突變或蛋白質的藥物 | Erlotinib、Trastuzumab、Imatinib |
| **免疫治療** | 活化免疫系統對抗癌細胞（含 ICI） | Pembrolizumab、Nivolumab、Atezolizumab |
| **荷爾蒙治療** | 抑制荷爾蒙依賴性癌症生長 | Tamoxifen、Letrozole、Bicalutamide |

## 8. 院內病歷互審系統

> 彰濱秀傳癌症中心個管師病歷照護品質互審平台

---

### 一、登入

使用你自己的帳號和密碼登入。如忘記帳號密碼，請洽系統管理員。

登入後依你的角色會看到不同的功能選單：

| 角色 | 可使用功能 |
|------|-----------|
| **個管師（case_manager）** | 日常作業、審核派送、輔助資源 |
| **Admin** | 以上全部 + 系統設定、系統管理、主檔管理 |
| **Reviewer（審核者）** | 審核介面（雲端端）|

---

### 二、系統概覽

病歷互審系統是一套**癌症個管照護品質互審平台**，核心流程如下：

```
個管師選取病歷 → 建立審核批次 → 派送給雲端 reviewer
→ Reviewer 線上審核 → 結果自動回填本地 → 個管師看報表
```

#### 主選單功能對照

| 選單 | 功能說明 |
|------|---------|
| **日常作業 → 病歷列表** | 查詢、標記、管理所有病人病歷 |
| **日常作業 → 審核派送** | 建立審核批次、派送給 reviewer、收回結果 |
| **審核輔助資源** | 審核標準說明書、評分表參考 |
| **主檔管理** | 癌別、MDT、標記原因、科別醫師（Admin）|
| **系統管理** | 備份、雲端同步、主檔載入（Admin）|

---

### 三、病歷列表

#### 3-1 查詢病歷

進入「日常作業 → 病歷列表」。

- 上方搜尋列輸入**姓名**或**病歷號**即時篩選
- 可依**癌別**、**MDT 群組**、**標記狀態**篩選
- 多重癌病人會顯示**「多重癌」**紅色標籤

#### 3-2 標記病歷（必抽 / 必不抽）

點擊病歷右側的標記按鈕，選擇標記原因：

| 標記類型 | 說明 |
|---------|------|
| **必抽** | 本筆病歷下次建批次時必須納入審核 |
| **必不抽** | 本筆病歷排除於抽樣之外 |
| 無標記 | 系統隨機抽樣決定 |

#### 3-3 個別病歷詳情

點擊任一病歷進入詳情頁，可查看：

- 基本資訊（癌別、分期、主治醫師、MDT 群組）
- 歷次審核記錄（哪個批次被審過、reviewer 給分）
- 標記歷程

---

### 四、審核批次管理

#### 4-1 建立批次

1. 進「日常作業 → 審核派送」
2. 右上角點「**+ 建立批次**」
3. 填寫：
   - 批次名稱（建議格式：`2026Q2-乳癌`）
   - 審核截止日期
   - 選擇 MDT 群組（決定抽哪些癌別的病歷）
   - 抽樣比例（系統依比例隨機抽樣）
4. 點「建立」→ 系統自動抽樣完成

!!! note "必抽 / 必不抽優先"
    標記「必抽」的病歷一定會被選入；標記「必不抽」的病歷一定不會被選入，不受抽樣比例影響。

#### 4-2 批次列表

批次列表會顯示「**同步狀態**」欄，一眼看完所有派送進度：

| 狀態 Badge | 意思 | 該做什麼 |
|-----------|------|---------|
| 🟡 未派送 | 批次建好但還沒送出 | 進詳情頁點「派送雲端」|
| 🔵 已派送 | 已送給 reviewer，審核中 | 等（或手動「立即同步」）|
| 🔵 部分收回 3/5 | 部分 reviewer 已完成 | 等其餘 reviewer |
| 🟢 已收回 5/5 | 全部完成 | 看品質報表 |

#### 4-3 派送雲端

點進批次詳情頁 → 右上角「**☁️ 派送雲端**」→ 系統彈出**預覽 Modal**：

**預覽 Modal 內容（由上而下）：**

1. **環境檢查**：確認雲端連線設定是否正確
2. **既有 Outbox 警告**：若此批次曾派送失敗，會顯示歷史錯誤訊息
3. **批次摘要**：批次名稱、截止日、將派送筆數、Conflict 數
4. **Conflict 列表**：若 reviewer 是該病歷主治醫師（利益迴避），系統自動列出，需先解決才能派送
5. **派送內容預覽 5 筆**：顯示真名 vs. 雲端遮蔽名對照（確認去識別化正確）

確認無誤後點「**✓ 確認派送**」完成派送。

#### 4-4 收回審核結果

系統每 **30 分鐘**自動收回一次。也可手動：

- **單批次**：進批次詳情頁 → 點「☁️ 收回審核結果」
- **全部批次**：系統管理 → 雲端同步 → 點「🔄 立即同步」

---

### 五、雲端同步監控（Admin）

進「系統設定 → 系統管理 → ☁️ 雲端同步」Tab：

#### 5 大區塊

| 區塊 | 說明 |
|------|------|
| **環境檢查** | 確認雲端設定是否正常 |
| **摘要卡** | 總派送批次、已收回、同步中、Outbox 待重試、需要介入 |
| **派送批次列表** | 每批次完成度進度條 + 操作按鈕 |
| **需要介入（Dead）** | 重試 10 次仍失敗的工作，需 Admin 手動 retry |
| **Outbox 待重試** | 正在自動重試中的工作 |

#### 失敗重試機制

派送/收回失敗時系統自動用**指數退避**重試：

```
失敗 → 1分後重試 → 5分 → 30分 → 1小時 → 4小時 → 12小時 → 24小時
重試 10 次仍失敗 → 標記為 Dead（需 Admin 介入）
```

Dead 紀錄處理：確認問題 → 修好後點「🔄 retry」 → 仍無法則點「🗑 delete」放棄。

### 八、常見問題

**Q：換版後登入失敗？**
A：重新整理頁面後再以自己的帳號密碼登入。若仍無法登入，請洽系統管理員。

**Q：我的批次一直顯示「部分收回」，不會變成全部收回？**
A：有 reviewer 尚未完成審核。進批次詳情頁查看哪位 reviewer 的進度還未完成，可聯繫提醒。

**Q：派送時出現 Conflict 警告，無法送出？**
A：系統偵測到 reviewer 是某筆病歷的主治醫師（利益迴避）。請回批次編輯頁，重新分配那幾筆的 reviewer，再進行派送。

**Q：30 分鐘過了，審核結果還沒回來？**
A：進「系統管理 → 雲端同步」查看是否有 Outbox 失敗紀錄。若有 Dead 紀錄，需 Admin 確認問題後手動 retry。

**Q：科別選單沒有影像診斷科/病理科？**
A：系統管理 → 📥 主檔載入 → 點「立即同步科別/醫師」。

**Q：想清除測試資料？**
A：系統管理 → 🐛 開發測試 → 「清除測試資料」按鈕。
